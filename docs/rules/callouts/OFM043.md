# OFM043 - Callout in list

**Severity:** warning
**Fixable:** no
**Added in:** Phase 5

## What it does

Warns when a callout directly follows a list item without a blank line
between them. Obsidian (and CommonMark in general) needs the blank
separator to render the block correctly; otherwise the callout becomes
part of the list item.

Bullet (`-`, `*`, `+`) and ordered list items are both recognised.

## Bad example

```markdown
- item
> [!NOTE] Title
> body
```

## Good example

```markdown
- item

> [!NOTE] Title
> body
```

## Config

Disable entirely:

```jsonc
{
  "rules": {
    "OFM043": { "enabled": false }
  }
}
```
