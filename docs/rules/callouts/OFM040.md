# OFM040 - Unknown callout type

**Severity:** error
**Fixable:** no
**Added in:** Phase 5

## What it does

Reports every `> [!TYPE]` callout whose type is not in the configured
`callouts.allowList`. Obsidian renders unknown types as plain quote
blocks with no visual feedback, which almost always signals a typo.

Case sensitivity follows `callouts.caseSensitive` (default: false).

## Bad example

With the default allowList:

```markdown
> [!CUSTOM] My callout
> body
```

## Good example

```markdown
> [!NOTE] My callout
> body
```

## Config

Extend the allowList:

```jsonc
{
  "callouts": {
    "allowList": ["NOTE", "WARNING", "CUSTOM"]
  }
}
```

Disable entirely:

```jsonc
{
  "rules": {
    "OFM040": { "enabled": false }
  }
}
```
