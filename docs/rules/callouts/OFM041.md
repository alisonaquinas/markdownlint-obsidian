# OFM041 - Malformed callout

**Severity:** error
**Fixable:** no
**Added in:** Phase 5

## What it does

Catches lines that look like they are trying to open a callout but do
not conform to the grammar used by Obsidian and the linter's own
extractor. Examples:

- `> [!NOTE]Title` - missing space after `]`
- `> [!] Title` - empty type
- `> [ NOTE ] Title` - stray spaces inside brackets

Well-formed callouts (including fold markers `+` and `-`) and plain
quote blocks are silently skipped. Malformed patterns inside fenced
code blocks are also skipped.

## Bad example

```markdown
> [!NOTE]Title
> body
```

## Good example

```markdown
> [!NOTE] Title
> body
```

## Config

Disable entirely:

```jsonc
{
  "rules": {
    "OFM041": { "enabled": false }
  }
}
```
