# OFM905 - Failed to load custom rule module

**Added in:** Phase 10

## What it does

Emitted to stderr when a path listed in `customRules` cannot be imported
(file not found, syntax error, or failed validation check). The
offending entry is skipped; all other rules still run.

## Example stderr output

```
OFM905: failed to load custom rule module "./my-rule.ts": Cannot find module
```

## Resolution

1. Confirm the path exists relative to the config file location.
2. Compile `.ts` sources to `.js` before running, or use `tsx`:
   `npx tsx packages/cli/bin/markdownlint-obsidian.js "**/*.md"`
3. Ensure the module exports a default `OFMRule` object (or array).
