# OFM904 - Duplicate custom rule skipped

**Added in:** Phase 10

## What it does

Emitted to stderr when a `customRules` entry exports a rule whose `names[0]`
is already registered (by a built-in rule or an earlier custom entry).
The duplicate is skipped; the run continues.

## Example stderr output

```
OFM904: skipped duplicate custom rule "OFM001": Duplicate rule name: OFM001
```

## Resolution

Give your custom rule a unique name that does not overlap any built-in OFM
or MD code. See the [[rules/index|rule catalog]] for the full reserved set.
