# OFM022 - Embed target missing

**Severity:** error
**Fixable:** no
**Added in:** Phase 5

## What it does

Probes the real filesystem via the injected `FileExistenceChecker` for
every non-markdown embed target. Images, PDFs, audio, and video assets
must exist beneath the detected vault root. Markdown embeds are handled
by OFM020; unknown extensions are handled by OFM024.

This is the first asynchronous rule in the codebase — `run` returns a
`Promise<void>` and `LintUseCase` awaits it.

When `resolve` is disabled this rule is a no-op (no vault root to probe
against).

## Bad example

```markdown
![[missing.png]]
```

## Good example

```markdown
![[assets/diagram.png]]
```

assuming `assets/diagram.png` exists beneath the vault root.

## Config

Disable this rule in `.obsidian-linter.jsonc`:

```jsonc
{
  "rules": {
    "OFM022": { "enabled": false }
  }
}
```
