# OFM020 - Broken embed (markdown)

**Severity:** error
**Fixable:** no
**Added in:** Phase 5

## What it does

Resolves every `![[target]]` embed whose target is a markdown file (or
extensionless) against the vault file index. Fires an error when the
target cannot be found. Non-markdown assets (images, PDFs, audio, video)
are handled by OFM022, not this rule.

When `resolve` is disabled (`--no-resolve` or `resolve: false`) this rule
is a no-op.

## Bad example

```markdown
![[missing-note]]
```

## Good example

```markdown
![[existing-note]]
```

## Config

Disable this rule in `.obsidian-linter.jsonc`:

```jsonc
{
  "rules": {
    "OFM020": { "enabled": false }
  }
}
```
