# OFM023 - Embed size invalid

**Severity:** warning
**Fixable:** no
**Added in:** Phase 5

## What it does

Warns when an embed's width or height sizing hint exceeds the caps
configured in `embeds.maxWidth` / `embeds.maxHeight`. Both caps are
`null` by default, so this rule is silent unless a project explicitly
opts in. Each axis is evaluated independently — a single embed may fire
twice if both width and height exceed their caps.

## Bad example

With `embeds.maxWidth: 400`:

```markdown
![[photo.png|800]]
```

## Good example

With `embeds.maxWidth: 400`:

```markdown
![[photo.png|400]]
```

## Config

```jsonc
{
  "embeds": {
    "maxWidth": 800,
    "maxHeight": 600
  }
}
```

Disable entirely:

```jsonc
{
  "rules": {
    "OFM023": { "enabled": false }
  }
}
```
