# OFM082 — Unknown top-level frontmatter key

**Severity:** warning
**Fixable:** no
**Added in:** Phase 3
**Default:** disabled

## What it does

Warns when a file's frontmatter contains a top-level key that is not
declared in `frontmatter.typeMap`. The rule is a no-op when
`frontmatter.allowUnknown` is `true` (the default), so vaults that have
not opted into a strict schema see no output.

## Bad example

```markdown
---
title: Note
weird_key: yes
---

Body.
```

With `typeMap = { title: "string" }` and `allowUnknown: false`,
`weird_key` is reported.

## Good example

```markdown
---
title: Note
---

Body.
```

## Config

```jsonc
{
  "frontmatter": {
    "typeMap": { "title": "string" },
    "allowUnknown": false
  },
  "rules": { "OFM082": { "enabled": true } }
}
```
