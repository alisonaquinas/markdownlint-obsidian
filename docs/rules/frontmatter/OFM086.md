# OFM086 — Frontmatter trailing whitespace

**Severity:** warning
**Fixable:** yes (Phase 9)
**Added in:** Phase 3

## What it does

Walks every string value reachable from `parsed.frontmatter`, including
nested maps and array elements, and warns when one ends with one or more
spaces or tabs. Designed to keep diffs clean and avoid YAML serialisation
surprises.

## Bad example

```markdown
---
title: "Note  "
---

Body.
```

## Good example

```markdown
---
title: "Note"
---

Body.
```

## Config

OFM086 requires no configuration.

## Known limitations

### Line attribution

Violations are attributed to the nearest **top-level** key's line number, not
to the exact line of a nested key. For example, a trailing-space value buried
under `author.bio` will report the line of `author:`, not of `bio:`.

### Keys with special characters

The key-line scanner uses a simplified regex that matches identifiers matching
`[A-Za-z0-9_-]`. Keys that contain characters outside this set — such as dots
(`a.b`), spaces, or Unicode — may fall back to reporting `line: 1` because
the scanner cannot locate the key reliably in the raw frontmatter text.
