# OFM122 - Malformed highlight

**Severity:** error
**Fixable:** no
**Added in:** Phase 6

## What it does

Reports lines with an odd number of `==` markers outside fenced code.
Each well-formed `==highlight==` span contributes two markers, so an odd
count is a reliable signal of an unterminated highlight.

The rule strips inline backtick code (``` `...` ```) before counting so
prose that references JavaScript's `===` operator does not trip the
check. Fenced code blocks are also skipped via the same state machine
that OFM041 uses.

## Bad example

```markdown
An unterminated ==highlight on a single line.
```

## Good example

```markdown
A well-formed ==highlight== on a single line.
```

## Known limitations

OFM122 does not track markdown-it level inline code positions; unusual
cases like `` `a` ==`b` ==`c` `` can still trip it. Teams that rely on
this pattern should disable OFM122 in their config.

## Config

Disable:

```jsonc
{
  "rules": {
    "OFM122": { "enabled": false }
  }
}
```
