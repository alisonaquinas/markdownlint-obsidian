# OFM124 - Empty highlight

**Severity:** warning
**Fixable:** yes (Phase 9)
**Added in:** Phase 6

## What it does

Reports highlights whose content is empty or whitespace-only. Obsidian
renders these as empty spans; they are almost always left over after a
refactor. Marked `fixable: true`; the autofixer lives in Phase 9 and
deletes the offending marker pair.

## Bad example

```markdown
Prose ==   == more prose.
```

## Good example

```markdown
Prose ==real highlight== more prose.
```

## Config

Disable:

```jsonc
{
  "rules": {
    "OFM124": { "enabled": false }
  }
}
```
