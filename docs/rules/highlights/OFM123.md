# OFM123 - Nested highlight

**Severity:** error
**Fixable:** no
**Added in:** Phase 6

## What it does

Reports lines that contain three `==` marker pairs. Obsidian cannot
render nested `==a ==b== c==` spans; the inner pair closes the outer one
and the trailing `==` becomes stray text.

The rule uses a conservative regex that fires on any line with three
`==` markers separated by non-`=` characters. It cannot distinguish
"nested" from "two adjacent highlights on one line" (`==a== and ==b==`),
so both trip the rule. Teams that rely on multiple highlights per line
should disable OFM123 or split into separate lines.

Inline backtick code is stripped before scanning so `==` operators inside
inline code do not contribute to the count.

## Bad example

```markdown
Outer ==a ==nested== inner== close.
```

## Good example

```markdown
Outer ==a== and a separate ==b==.
```

(still flagged under the known trade-off above — see "Known limitations")

## Known limitations

The rule does not understand highlight semantics; it only counts markers
per line. Teams that want multiple highlights per line should disable
OFM123.

## Config

Disable:

```jsonc
{
  "rules": {
    "OFM123": { "enabled": false }
  }
}
```
