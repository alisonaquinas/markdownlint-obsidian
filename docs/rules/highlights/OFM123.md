# OFM123 - Nested highlight

**Severity:** error
**Fixable:** no
**Added in:** Phase 6

## What it does

Reports lines that contain a truly nested highlight — where an opening `==`
marker is followed by content that ends with a space before the greedy-closing
`==`, indicating the author attempted to place one `==...==` span inside
another (e.g. `==outer ==inner== text==`).

Obsidian cannot render nested highlights. The inner `==` pair closes the outer
span early, and any trailing `==` becomes stray text or an unexpected second
highlight.

Adjacent highlights on the same line (`==a== and ==b==`) are **not** flagged;
each forms its own valid, non-overlapping span.

Inline backtick code is stripped before scanning so `==` operators inside
inline code do not contribute to the check.

## Bad example

```markdown
Outer ==a ==nested== inner== close.
```

## Good example

```markdown
==a== and ==b==
```

Two adjacent highlights on one line — both are valid and OFM123 does not fire.

## Config

Disable:

```jsonc
{
  "rules": {
    "OFM123": { "enabled": false }
  }
}
```
