# OFM120 - Disallowed highlight

**Severity:** error
**Fixable:** no
**Added in:** Phase 6
**Default:** disabled

## What it does

Reports every `==highlight==` when `config.highlights.allow` is `false`.
`allowedGlobs` is a per-glob escape hatch — a path matching any listed
glob is exempt, which lets teams ban highlights everywhere except (e.g.)
`notes/daily/**`.

The rule is disabled by default; flip `highlights.allow` to `false` in
`.obsidian-linter.jsonc` AND enable `rules.OFM120` to opt in.

## Bad example

With `highlights.allow: false`:

```markdown
An important ==highlight== in prose.
```

## Good example

```markdown
An important *emphasized* phrase in prose.
```

## Config

Ban everywhere:

```jsonc
{
  "highlights": { "allow": false, "allowedGlobs": [] },
  "rules": { "OFM120": { "enabled": true } }
}
```

Allow highlights only in the daily-notes subtree:

```jsonc
{
  "highlights": { "allow": false, "allowedGlobs": ["notes/daily/**"] },
  "rules": { "OFM120": { "enabled": true } }
}
```
