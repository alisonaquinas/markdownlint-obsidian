# OFM121 - Disallowed comment

**Severity:** error
**Fixable:** no
**Added in:** Phase 6
**Default:** disabled

## What it does

Two independent switches on `config.comments`:

- `allow: false`           → every `%%…%%` reports.
- `disallowMultiline: true` → only multi-line comments report.

The rule is disabled by default; flipping either flag is enough to opt
in — also enable `rules.OFM121`.

## Bad example

With `comments.allow: false`:

```markdown
Visible prose %% hidden annotation %% more prose.
```

With `comments.disallowMultiline: true`:

```markdown
Before %%
multi-line
block
%% after.
```

## Good example

```markdown
Visible prose with no comments.
```

## Config

Ban all comments:

```jsonc
{
  "comments": { "allow": false, "disallowMultiline": false },
  "rules": { "OFM121": { "enabled": true } }
}
```

Keep single-line comments, ban multi-line:

```jsonc
{
  "comments": { "allow": true, "disallowMultiline": true },
  "rules": { "OFM121": { "enabled": true } }
}
```
