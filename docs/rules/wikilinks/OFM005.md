# OFM005 - Wikilink case mismatch

**Severity:** warning
**Fixable:** yes (Phase 9)
**Added in:** Phase 4

## What it does

Warns when a wikilink only resolves thanks to case-insensitive fallback.
The canonical file spelling is reported so vaults that publish to
case-sensitive filesystems (e.g. Linux CI publishing to macOS) don't break
at publish time.

## Bad example

Vault contains `notes/Index.md`:

```markdown
See [[notes/index]] for details.
```

## Good example

```markdown
See [[notes/Index]] for details.
```

## Config

Control case sensitivity globally:

```jsonc
{
  "wikilinks": {
    "caseSensitive": true
  }
}
```
