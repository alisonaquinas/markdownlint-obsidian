# OFM007 - Wikilink block reference (deprecated)

**Severity:** error
**Fixable:** no
**Added in:** Phase 4
**Deprecated in:** Phase 6 — use [[rules/block-references/OFM102]] instead.

## What it does

OFM007 was a Phase 4 placeholder that only checked whether the target
*file* of a block-reference wikilink (`[[page#^blockid]]`) existed. Phase 6
ships the full cross-file check as OFM102, which also verifies that the
referenced block id is actually declared in the target file.

OFM007 is retained as an alias of OFM102 so vaults with existing
configuration keyed on `rules.OFM007` continue to work without a migration
step. New vaults should reference `OFM102` directly.

## Bad example

Vault contains `existing.md` but it does not declare `^b7`:

```markdown
See [[existing#^b7]] for the snippet.
```

## Good example

```markdown
See [[existing#^b7]] for the snippet.
```

(where `existing.md` contains a paragraph ending in `^b7`)

## Config

OFM007 takes no options. To disable the rule entirely, prefer:

```jsonc
{
  "rules": {
    "OFM102": { "enabled": false }
  }
}
```
