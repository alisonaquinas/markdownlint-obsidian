# OFM004 - Ambiguous wikilink target

**Severity:** error
**Fixable:** no
**Added in:** Phase 4

## What it does

Errors when a wikilink basename matches more than one `.md` file in the
vault. The resolver cannot decide which file to link to, so the correct fix
is to spell out the full relative path.

## Bad example

Vault contains `notes/index.md` and `archive/index.md`:

```markdown
See [[index]] for details.
```

## Good example

```markdown
See [[notes/index]] for details.
```

## Config

OFM004 takes no options.
