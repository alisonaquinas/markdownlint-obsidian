# MD018 — no-missing-space-atx

**Why disabled in markdownlint-obsidian.** Obsidian's nested tag syntax
(`#project`, `#area/notes`) starts with a literal `#` followed
immediately by the tag name. A line beginning with `#project` is a
perfectly valid body-level tag, but markdownlint sees `#` + word and
reports it as a malformed ATX heading missing its space. The OFM tag
family (`OFM060`–`OFM066`) already validates the real tag syntax, so
MD018 is pure noise in any vault that uses line-leading tags.

## Re-enable it in your own config

If your vault never starts a line with a tag, you can re-enable the rule
safely:

```jsonc
{
  "rules": {
    "MD018": { "enabled": true }
  }
}
```

Note that re-enabling MD018 will fire on every line that begins with a
`#tag`, regardless of whether the tag is valid OFM. There is no
upstream markdownlint option that distinguishes tags from malformed
headings — that's what makes this a conflict rather than a misfire.
