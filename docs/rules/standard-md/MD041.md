# MD041 — first-line-heading

**Why disabled in markdownlint-obsidian.** Obsidian vaults frequently
contain frontmatter-only notes: a `---` YAML block with metadata keys,
followed by body prose that does not start with a heading. MD041
demands that the first non-frontmatter line be a top-level `#` heading,
which would reject every such note on sight.

The OFM frontmatter rule family (`OFM080`–`OFM087`) already enforces
frontmatter hygiene; there's no value in layering a heading-shape rule
on top of it.

## Re-enable it in your own config

If your vault follows a strict "title-first" convention, you can turn
MD041 back on. The upstream rule accepts a `level` override if your
titles live at `##` instead of `#`:

```jsonc
{
  "rules": {
    "MD041": {
      "enabled": true,
      "options": { "level": 1 }
    }
  }
}
```
