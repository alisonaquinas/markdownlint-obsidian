# MD033 — no-inline-html

**Why disabled in markdownlint-obsidian.** Obsidian flavoured markdown
renders several constructs through inline HTML: raw `<details>` blocks
for collapsible sections, `<br>` line breaks inside callout titles,
embedded SVG figures, and the occasional `<sup>` for footnote markers.
All of these are first-class Obsidian features, so a global "no inline
HTML" rule produces false positives on day-one content.

## Re-enable it in your own config

If you publish your vault through a pipeline that disallows raw HTML,
re-enabling the rule with an allow-list is the usual compromise:

```jsonc
{
  "rules": {
    "MD033": {
      "enabled": true,
      "options": {
        "allowed_elements": ["details", "summary", "br", "sub", "sup"]
      }
    }
  }
}
```
