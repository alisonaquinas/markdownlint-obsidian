# MD028 — no-blanks-blockquote

**Why disabled in markdownlint-obsidian.** OFM callouts use the blockquote
fence (`>`) as their container syntax. Multi-paragraph callouts *require* a
blank `>` line between paragraphs to keep each paragraph inside the callout:

```markdown
> [!NOTE]
> First paragraph of the callout.
>
> Second paragraph — the blank `>` line above is mandatory.
> Without it Obsidian ends the callout and renders the rest as plain text.
```

MD028 fires on every one of those separator lines. Because the blank
`>` lines are structurally required by OFM callout rendering — not merely
stylistic — the rule produces only false positives in any vault that uses
multi-paragraph callouts.

## Re-enable it in your own config

If your vault never uses multi-paragraph callouts (or uses none at all) you
can re-enable the rule:

```jsonc
{
  "rules": {
    "MD028": { "enabled": true }
  }
}
```

Be aware that re-enabling MD028 will flag every blank `>` separator line
inside callouts, including ones that are required for correct rendering.
There is no upstream markdownlint option that distinguishes OFM callout
separators from genuinely accidental blank blockquote lines — that's what
makes this a conflict rather than a misfire.
