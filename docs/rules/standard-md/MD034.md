# MD034 — no-bare-urls

**Why disabled in markdownlint-obsidian.** Obsidian auto-links bare URLs
in body text by default: a line containing `See https://example.com for
details.` renders identically to one that wraps the URL in explicit
angle brackets. Forcing users to wrap every bare URL would be pure
friction with no rendering benefit on the most common Obsidian reader
surfaces.

## Re-enable it in your own config

If you publish through a pipeline that produces output for environments
where auto-linking is unreliable (plain `.txt` exports, some static
site generators), re-enable the rule:

```jsonc
{
  "rules": {
    "MD034": { "enabled": true }
  }
}
```

The upstream fix wraps bare URLs in `<…>` brackets, which Obsidian
still renders correctly — so there's no content-destructive downside
to flipping it on.
