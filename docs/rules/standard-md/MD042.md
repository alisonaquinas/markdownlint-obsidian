# MD042 — no-empty-links

**Why disabled in markdownlint-obsidian.** markdownlint parses Obsidian
wikilinks `[[target]]` as two nested empty bracketed links: an outer
`[…]` containing an inner `[…]`. The inner link has no URL target, so
MD042 reports it as "empty link". Every wikilink in the vault would
trip the rule.

The OFM wikilink family (`OFM001`–`OFM007`) already validates wikilink
syntax and target resolution — they're the rules users actually want
for link hygiene in an Obsidian vault.

## Re-enable it in your own config

If you have a subset of files that use only standard markdown links
(no wikilinks), you can re-enable MD042 globally and rely on explicit
ignores for the wikilink-heavy areas:

```jsonc
{
  "rules": {
    "MD042": { "enabled": true }
  },
  "ignores": ["vault/notes/**"]
}
```

There is no upstream option that teaches MD042 about wikilink syntax.
