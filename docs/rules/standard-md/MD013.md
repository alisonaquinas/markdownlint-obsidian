# MD013 — line-length

**Why disabled in markdownlint-obsidian.** Obsidian wikilinks and embeds
(`[[very/long/note-name#heading|alias]]`, `![[path/to/image.png|400]]`)
routinely produce single lines longer than markdownlint's 80-column
default. Forcing a wrap on those constructs either breaks the link
syntax or introduces awkward mid-link line splits that many Obsidian
clients then render incorrectly.

## Re-enable it in your own config

The safest way to opt back in is to lift the default limit to a value
that matches your vault's column style. 120 is a common choice:

```jsonc
{
  "rules": {
    "MD013": {
      "enabled": true,
      "options": { "line_length": 120 }
    }
  }
}
```

For vaults that prefer hard 80-column prose but still need wikilinks to
be exempt, use markdownlint's per-kind toggles:

```jsonc
{
  "rules": {
    "MD013": {
      "enabled": true,
      "options": {
        "line_length": 80,
        "code_blocks": false,
        "tables": false
      }
    }
  }
}
```
