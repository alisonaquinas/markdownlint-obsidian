# OFM103 - Block reference on heading

**Severity:** warning
**Fixable:** no
**Added in:** Phase 6

## What it does

Warns when `^blockid` is attached to a heading line. Obsidian renders
headings as their own navigation targets, and a trailing block id on a
heading line is silently ignored — authors almost always intended either
a normal paragraph anchor below the heading or a heading link
(`[[page#Heading]]`).

## Bad example

```markdown
# Introduction ^intro

Body text.
```

## Good example

```markdown
# Introduction

Body text. ^intro
```

## Config

Disable entirely:

```jsonc
{
  "rules": {
    "OFM103": { "enabled": false }
  }
}
```
