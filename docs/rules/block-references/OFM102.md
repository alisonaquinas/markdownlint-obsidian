# OFM102 - Broken block link

**Severity:** error
**Fixable:** no
**Added in:** Phase 6

## What it does

Validates `[[page#^blockid]]` wikilinks against the cross-file
`BlockRefIndex` built during vault bootstrap. Supersedes the Phase 4
placeholder [[rules/wikilinks/OFM007]], which only checked that the
*target file* existed. OFM102 additionally verifies the referenced block
id is actually declared inside that file.

Skips links whose target page does not resolve (OFM001 already reports
those) and links that have no block-reference component.

## Bad example

Vault contains `notes/a.md` with content `# A`, no block ids declared:

```markdown
See [[a#^intro]] for context.
```

## Good example

`notes/a.md`:

```markdown
# A

Paragraph with the block anchor ^intro
```

`notes/b.md`:

```markdown
See [[a#^intro]] for context.
```

## Config

Disable entirely:

```jsonc
{
  "rules": {
    "OFM102": { "enabled": false }
  }
}
```

`OFM007` is kept as a back-compat alias of this rule; either identifier
resolves to the same behaviour.
