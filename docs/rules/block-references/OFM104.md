# OFM104 - Block id case

**Severity:** warning
**Fixable:** yes (Phase 9)
**Added in:** Phase 6

## What it does

Style warning that surfaces block ids containing uppercase letters.
Obsidian is case-sensitive on block-id resolution, so mixing `^Intro`
and `^intro` produces subtle "wrong anchor" bugs; keeping every id
lowercase removes an entire class of oversight.

Marked `fixable: true`; the autofixer lives in Phase 9 and replaces the
identifier with its lowercase equivalent.

## Bad example

```markdown
Paragraph ^Intro
```

## Good example

```markdown
Paragraph ^intro
```

## Config

Disable entirely:

```jsonc
{
  "rules": {
    "OFM104": { "enabled": false }
  }
}
```
