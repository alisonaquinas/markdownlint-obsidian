# OFM100 - Invalid block reference

**Severity:** error
**Fixable:** no
**Added in:** Phase 6

## What it does

Reports every `^blockid` whose identifier fails the regex in
`config.blockRefs.idPattern`. Obsidian's own block-id grammar allows
letters, digits, and hyphens, and the default pattern mirrors that plus a
32-character cap so tests have a stable target and sorted ids stay
grep-friendly.

## Bad example

With the default pattern `^[A-Za-z0-9-]{1,32}$`:

```markdown
Paragraph ^this-block-id-is-far-too-long-and-will-be-reported
```

## Good example

```markdown
Paragraph ^intro-1
```

## Config

Override the pattern:

```jsonc
{
  "blockRefs": {
    "idPattern": "^[a-z0-9-]+$",
    "requireUnique": true
  }
}
```

Disable entirely:

```jsonc
{
  "rules": {
    "OFM100": { "enabled": false }
  }
}
```
